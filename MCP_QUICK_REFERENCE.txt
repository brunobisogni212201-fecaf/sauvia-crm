🔐 AUTHENTICATION FLOW
─────────────────────────────────────────────────────────────────────────────

1. USER LOGIN (Clerk)
   ├─ User signs up/in at app.sauvia.com
   └─ Clerk issues JWT with sub, email, org_id claims

2. MCP CLIENT CONNECTS (e.g., Claude)
   ├─ Discovers /.well-known/oauth-authorization-server
   ├─ Discovers /.well-known/oauth-protected-resource/mcp
   ├─ Initiates OAuth flow → redirects to Clerk
   └─ Clerk grants OAuth token back to MCP client

3. MCP TOOL EXECUTION
   ├─ MCP client sends tool request: /mcp/http
   ├─ Includes OAuth token in Authorization header
   ├─ Next.js middleware (withMcpAuth) verifies token
   ├─ Extracts Clerk claims → passed to tool handler
   └─ Tool calls backend API with token

4. BACKEND VERIFICATION
   ├─ FastAPI receives request with Bearer token
   ├─ clerk_auth.verify_clerk_token() validates RS256 signature
   ├─ Fetches Clerk JWKS if not cached
   ├─ Compares key ID (kid) and verifies signature
   ├─ Returns decoded user claims
   └─ Route proceeds with user context

🔗 INTEGRATION POINTS
─────────────────────────────────────────────────────────────────────────────

Backend routers need to include:
  from fastapi.security import HTTPBearer, HTTPAuthCredentials
  from src.adapters.driven.security.clerk_auth import verify_clerk_token

  security = HTTPBearer()

  @router.get("/protected")
  async def protected_route(credentials: HTTPAuthCredentials = Depends(security)):
      user = verify_clerk_token(f"Bearer {credentials.credentials}")
      return {"user_id": user.get("sub")}
